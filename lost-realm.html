CASE 03 / 3D STEALTH PUZZLE ADVENTURE

LOST REALM

类型: 3D 潜行解谜冒险

重点: 地下堡垒 / 机关谜题 / 潜行战斗 / UE 蓝图

工具: Unreal Engine / 白模迭代 / Blueprint 交互

RELIC

INTRODUCTION

这是一个把潜行、机关解谜和空间推进真正揉在一起的 3D 冒险关卡。

《Lost Realm》是一款 3D 潜行解谜项目。玩家会进入一座表面像军事地下堡垒、 深处却藏着古老宫殿气质的空间里，让探索、潜行、交互机关和叙事线索一起推动前进。

这个项目的重点不只是“解开谜题”，而是让玩家在仓库、走廊、展览厅、宫殿和地下区域之间逐步感受到空间推进、 节奏变化和目标不断被重新定义的过程。

BACKGROUND

寻宝者与失落文明

主角被设定成对古代文明着迷的寻宝者埃里克，他会追随亚特兰蒂斯的线索，进入希尔瓦斯森林上方废墟之下的地下要塞。

这个背景一方面给潜行和机关解谜提供了合理容器，另一方面也让后续的宫殿、雕像和红色水晶球这些视觉元素有了叙事支点。

REFERENCE

参考与角色设定

这个项目的灵感来自《神秘海域》系列、《最后生还者》和《耻辱 2》的 Clockwork Mansion 关卡。

项目希望同时具备遗迹探险的吸引力，并让机关、敌人巡逻和空间结构彼此咬合，形成稳定的关卡语言。

GAME MECHANICS

潜行、机关和文本线索被一起设计成系统，而不是彼此割裂。

A1

潜行机制

项目通过蹲伏、视线规避、无声移动和背刺时机来建立潜行节奏，让敌人巡逻成为空间阅读的一部分。

B2

机关谜题

拉杆、电池、电池盒、旋转地面和按钮共同构成多步解谜，重点在于逻辑关联和重复利用已学会的知识。

C3

文本与线索

书本、笔记和散落文本负责补充机关解释和世界细节，让空间不只是功能性的，也能承担叙事表达。

MAP & FLOW

地图推进围绕 Introduce、Practice、Master、Twist 四个节奏阶段展开。

INTRODUCE

入口区

先通过教学式谜题介绍拉杆、电池、电池盒和基础潜行，让玩家理解交互逻辑再进入主体空间。

PRACTICE

一楼与 Z 字走廊

这里把玩家从机关解谜过渡到潜行战斗，通过双敌人和折线路径训练视线规避与行动时机。

MASTER / TWIST

二楼、大厅与地下区域

后半段要求玩家综合使用前面学到的所有系统，并通过 Boss 式空间变化完成最终收束。

SECOND FLOOR

二楼与展览厅挑战

二楼被做成一个兼顾战斗、探索和机关观察的区域。餐厅负责潜行节奏，展览厅负责图案与机关配合， 可选房间则补充背景和奖励。

这个部分的目标不是单纯增加内容量，而是让玩家在“已经掌握系统”的前提下真正开始主动组合它们。

UNDERGROUND AREA

地下区域与双电池谜题

地下区域被设计成前面知识的重组场。玩家需要重新调用早期区域里的电池与电梯逻辑，才能解开最终路径。

这样能让空间显得更真实，因为先前的系统不是被一次性消耗掉，而是在结尾重新变成关键答案。

DESIGN DETAILS

从白模到空间叙事

入口走廊、仓库、展览厅和宫殿空间都在逐步强化“从冷色机械区到暖色遗迹区”的过渡，让玩家在视觉上也感受到目标越来越接近。

Z 字走廊和大空间转场的作用，不只是延长路径，而是控制视线、保持神秘感，并重新安排节奏。

ITERATION

地图与氛围的多轮调整

地图体量、出口位置、餐厅区域形状和中央目标物都经过了反复调整，让关卡既更合理，也更能支持潜行玩法。

把圆盘替换成巨大雕像之后，空间叙事和玩家目标变得更明确，整体记忆点也更强。

BLUEPRINTS

UE 蓝图交互系统

因为关卡里有很多可交互的机械物体，所以蓝图是这个项目非常核心的一部分。拉杆、电池、电池盒和书籍等交互被统一接入接口逻辑， 让玩家靠近时能获得清晰反馈。

对拉杆这种物体，还加入了是否需要电池、是否为一次性使用和动画完成状态等判断，保证系统在复杂场景里也能稳定工作。

PROBLEM SOLVING

从 Unity 经验迁移到 Unreal

这是第一次把完整 3D 关卡和大量交互逻辑放进 Unreal 里，因此最大的挑战之一，是把原本熟悉的 Unity 思路翻译成 UE 的工作方式。

在这个过程中，一边摸索 Blueprint 的结构，一边验证哪些系统更适合线性关卡表达。最终确认，线性推进比最初尝试的半开放布局更有效。
